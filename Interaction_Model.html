交互模型

Xiao AI Voice Service (简称AIVS)云端API兼容Alexa Voice Service v20160207(简称AVS)。为了更好理解交互模型，可以参考：

  • Interaction Model

Directive and Event

AIVS 云端API 是一系列细粒度接口的集合，每一个接口都使用Event/Directive的形式进行客户端/服务端交互。

  • Event为客户端像服务端发送的消息。通常为请求服务端处理某种事件或通知服务端某种事件发生。例如语音识别请求或文本识别请求。

  • Directive为服务端发往客户端的消息。通常为客户端请求的处理结果或用来指示客户端进行某种操作。例如下发音频的Url，指示客户端播放音频。

注意 针对客户端上报的一个Event，服务端可能下发多个Directives以要求客户端执行一系列动作。此时多个Directives处于Multipart 消息的不同Entity中，每个Entity只携带一个Directive。而不是一个Entity携带多个Directives的列表。

Event和Directive中都需要携带Header字段来指明对应的接口。目前AIVS 支持的接口包括：

接口 描述
SpeechRecognizer 音频识别接口，提供音频语义识别和理解的功能。通常在客户端上报的Event中指明。
SpeechSynthesizer 语音合成接口，返回AIVS合成的语音。 通常在服务端下发的Directive中指明。
TextRecognizer 语义理解接口，提供文本语义识别和理解的功能。通常在客户端上报的Event中指明。
AudioPlayer 音频播放接口，要求客户端播放音频资源。 通常在服务端下发的Directive中指明。
TemplateRuntime 展示模板接口，要求客户端按照模板展示。 通常在服务端下发的Directive中指明。

Header

Event和Directive都需要携带Header部分。Header中各字段意义如下：

namespace

指明调用接口。

name

动作类型。不同接口可以执行的动作类型不同。

messageId

消息Id，唯一标识一条请求。服务端下发的Directive中的messageId应当与对应的Event保持一致。客户端使用此字段来匹配请求和响应。

dialogRequestId

对话Id，多轮对话时用来区别一轮对话。一次请求如果需要多轮交互，客户端需要保持dialogRequestId一致，以表明仍然处于一次请求中，而不是新的请求。

交互流程

一次交互流程通常来说由客户端发起。客户端向服务端发起请求，服务端处理后将处理结果和要求客户端执行的操作通过Directives下发。客户端收到响应后执行下发的操作，并根据需要决定是否需要发起多轮交互。

一个典型的流程如下面示例。在这个例子中用户通过客户端请求播放《三国演义》：

  1. 客户端向AIVS服务端发送播放三国演义的请求。 请求为一条Multipart消息，其中JSON Entity部分为：

    {
 "event":{
     "header":{
         "namespace":"TextRecognizer",
         "name":"TextRecognize",
         "messageId":"a78c3f56-db9a-426f-9bab-933cde549d13",
         "dialogRequestId":"24979d8e-c251-4dad-9d85-f074fe3cf0c5"
     },
     "payload":{
         "text":"播放三国演义"
     }
 }
}

    header中的TextRecognizer，TextRecognize指明请求发送给语义识别接口，要求执行文本识别动作。payload部分携带了请求的文本信息“播放三国演义”。 由于是文本识别，消息不需要携带音频流部分。

  2. 服务端接收请求，处理后返回处理结果及需要客户端执行的操作。 服务端响应为一条Multipart消息。包括三个JSON Entities，每个JSON Entity包含一个Directive：

    {
 "directive":{
     "header":{
         "namespace":"SpeechSynthesizer",
         "name":"Speak",
         "messageId":"a78c3f56-db9a-426f-9bab-933cde549d13",
         "dialogRequestId":"24979d8e-c251-4dad-9d85-f074fe3cf0c5"
     },
     "payload":{
         "url":"cid:5ab32280-260c-4b05-b896-956126e910d5",
         "remoteUrl":"http://file.ai.xiaomi.com/v2/anonymous/file/speech_tts_mercury_93024c8159407b4cbd30680218c3f2b6_cc4623c1422b1eaf84d0079f7996ea9b.mp3",
         "format":"AUDIO_MPEG",
         "token":"5ab32280-260c-4b05-b896-956126e910d5",
         "text":"好的，小爱为你播放三国演义 第001回"
     }
 }
}

    header中的 SpeechSynthesizer，Speak 表明需要客户端播放直接播放音频。客户端可以通过payload->remoteUrl获取音频内容。

    {
 "directive":{
     "header":{
         "namespace":"AudioPlayer",
         "name":"Play",
         "messageId":"a78c3f56-db9a-426f-9bab-933cde549d13",
         "dialogRequestId":"24979d8e-c251-4dad-9d85-f074fe3cf0c5"
     },
     "payload":{
         "playBehavior":"REPLACE_ALL",
         "audioItem":{
             "audioItemId":"qqfm:rd003ADbAb0bOuNq",
             "stream":{
                 "url":"http://files.ai.xiaomi.com/aiservice/qqfm/bf8e7da91079a7c971ab8e3a47344866e91662de44cc72117800363133b90311.mp3",
                 "offsetInMilliseconds":0,
                 "progressReport":{
                     "progressReportDelayInMilliseconds":15000,
                     "progressReportIntervalInMilliseconds":900000
                 },
                 "token":"rd003ADbAb0bOuNq"
             }
         }
     }
 }
}

    header中的 AudioPlayer，Play 表明需要客户端播放音频资源。客户端可以通过payload->audioItem->stream->url获取音频内容。

    {
 "directive":{
     "header":{
         "namespace":"TemplateRuntime",
         "name":"RenderPlayerInfo",
         "messageId":"a78c3f56-db9a-426f-9bab-933cde549d13",
         "dialogRequestId":"24979d8e-c251-4dad-9d85-f074fe3cf0c5"
     },
     "payload":{
         "audioItemId":"qqfm:rd003ADbAb0bOuNq",
         "content":{
             "title":"第001回",
             "header":"三国演义",
             "mediaLengthInMilliseconds":0,
             "art":{
                 "sources":[
                     {
                         "url":"http://imgcache.qq.com/fm/photo/album/rmid_album_360/H/W/0046ZrNH32e4HW.jpg?time=1511154355",
                         "widthPixels":0,
                         "heightPixels":0
                     }
                 ]
             },
             "provider":{
                 "name":"qqfm"
             }
         },
         "controls":[
             {
                 "type":"BUTTON",
                 "name":"NEXT",
                 "enabled":true,
                 "selected":false
             },
             {
                 "type":"BUTTON",
                 "name":"PREVIOUS",
                 "enabled":true,
                 "selected":false
             },
             {
                 "type":"BUTTON",
                 "name":"PLAY_PAUSE",
                 "enabled":true,
                 "selected":false
             }
         ]
     }
 }
}

    header中的 TemplateRuntime，RenderPlayerInfo 表明需要客户端按照模板展示内容。

  3. 客户端收到响应后依次执行服务端下发的操作：

    3.1. 播放Speak Directive中的音频。

    3.2. 播放Play Directive中的音频资源。

    3.3. 在屏幕上按照模板展示渲染后的内容。

  4. 客户端判断不需要进一步交互。此次交互结束。
