1.云端API对接-服务端Demo

本Demo受限于服务端开发环境限制，无法采集音频声音，目前暂提供TextRecognizer.TextRecognize的演示。Demo下载地址请联系具体对接人员

1.1 运行方法

在运行之前，您需要修改src/main/resources/config/aivsclient.conf文件，填写appId,deviceId,accessToken,如下所示

com.xiaomi.aivoice.aivsclient {
  default {
    appId = "" // 填写应用程序的appId
    deviceId = "" // 填写当前设备的deviceId
    accessToken = "" // 填写通过小米账号OAuth获得的access_token, 详细请参考授权码授权模式https://dev.mi.com/console/doc/detail?pId=707
  }
}

在服务端Demo工程下运行如下命令

// HTTP2 请求，建议使用该接入方式
gradle task testHttp2

如果本地没有安装gradle，可以尝试在工程目录下运行如下命令

./gradlew task testHttp2

// HTTP1.1 请求
gradle task testHttp1_1

如果本地没有安装gradle，可以尝试在工程目录下运行如下命令

./gradlew task testHttp1_1

1.2 Demo版本更新

v1.1 使用accessToken替代原有token
v1.0 提供TextRecognizer.TextRecognize的Demo，包括发送request和解析response请求。

2.云端API对接-客户端Demo

本Demo完整演示与AIVS交互的整体流程，包括可视化显示、播放接口列表中的所有功能。Demo下载地址请联系具体对接人员

2.1 运行方法

  1. 根据开发环境，分别阅读alexa-avs-sample-app对应的文档，启动该客户端Demo。 例如开发环境为Windows，那么请阅读Windows环境 ，只需要阅读3-Install dependencies、6-Run the sample APP、7-Talk to Alexa

  2. 在目前的接入方式中，暂时需要开发人员手动点击界面Login按钮，在Access token中填入对接人员提供的token。
