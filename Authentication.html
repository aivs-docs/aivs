消息鉴权

访问Xiao AI Voice Service 云端API的请求应当携带鉴权信息。 根据接入方式不同,API可以选择两种携带鉴权信息的方式, Header中携带或Query Parameters中携带。

关于接入方式,可以参考接入部分。

1. Http Header携带鉴权信息

如果客户端使用设备OAuth方式接入,可以使用Header携带鉴权信息方式。 此时在客户端每次向云端API发送请求时,都要在Http Header中携带 authorization字段。

authorization字段的格式为

d~{app_id}~{device_id}~{access_token}

需要填写的参数包括

app_id

应用注册的Id。需要和申请权限时的填写保持一致。

device_id

调用接口的设备的Id。需要和申请权限时的填写保持一致。 注意：设备Id请不要带有～符号。 

access_token

申请权限成功后下发的访问令牌。

示例

authorization: d~942339849586~8601177500456236~n8LHyTHufLE_0NwnLoV3s0AfLWT3HyuT4pplRSWpO4IYSI

2. Http Query Parameters携带鉴权信息

如果客户端使用 应用匿名方式接入,需要在Http query parameters中携带鉴权信息。(设备OAuth方式接入也可以使用query parameters携带鉴权信息)

需要填写的参数包括

auth_type

接入方式。可选值包括device_oauth(设备OAuth)和app_anonymous(应用匿名)

app_id

应用注册的Id。需要和申请权限时的填写保持一致。

device_id

调用接口的设备的Id。需要和申请权限时的填写保持一致。

api_key

小爱开发平台上配置的API Key。如果使用设备匿名方式接入,需要携带此字段。需要和申请权限时的填写保持一致。

token

申请权限成功后下发的访问令牌

示例

?auth_type=device_oauth&app_id=942339849586&device_id=8601177500456236&token=n8LHyTHufLE_0NwnLoV3s0AfLWT3HyuT4pplRSWpO4IYSI
