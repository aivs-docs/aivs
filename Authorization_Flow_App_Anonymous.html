应用匿名授权接入

如果用户没有小米账号，也可以使用应用匿名的方式使用Xiao AI Voice Service 服务。本文描述如何启用应用匿名方式接入,以及典型的授权流程。

启用应用匿名方式接入

启用应用匿名方式接入前，请确认您已接入小米账号。暨：

  1. 在小爱开放平台成功注册企业账号。
  2. 已经成功创建应用程序。

启用应用匿名方式接入需要作如下操作：

  1. 在应用的安全配置中的的回调地址。配置API Key。注意填写的MD5和SHA256字段都要有冒号分隔。

配置成功后小爱开放平台会分配API Key

授权流程

  1. 用户启动车辆。
  2. 车辆连接车厂后台服务，通知车辆启动。需要携带车辆设备Id。
  3. 车厂后台服务向小爱账号服务申请应用匿名接入令牌。请求中应当携带应用Client ID，应用SignSecret，车辆device_id，小爱开放平台中配置的MD5、SHA256，分配的API Key，grant_type以及使用上述字段计算出的签名字段。详见后面申请令牌部分。
  4. 小爱账号服务验证请求中的信息。
  5. 小米账号令牌中心验证通过后，返回接入令牌及刷新令牌。
  6. 车厂后台服务应当定时维护接入令牌和刷新令牌。保证令牌有效。 注意：除了车辆device_id，其他应用相关信息应保存在车厂后台服务，以避免信息泄露。车辆和车厂后台服务间通信的安全由车厂保证。

申请令牌接口

请求

Host： account.ai.xiaomi.com

Path： /anonymous/app/auth/token

Method：POST

Scheme：https

Parameters：

参数 描述
grant_type 应当填写为app_token，表明申请应用匿名接入令牌。
client_id 小爱开放平台中注册的应用Id。
api_key 加密后的小爱开放平台中分配的API Key。具体算法见下文。
device_id 车辆设备ID，应当保证每辆车唯一。
md5_sign 小爱开放平台中为API Key配置的MD5。
sha256_sign 小爱开放平台中为API Key配置的SHA256。
signature 使用以上字段及应用SignSecret字段产生的签名信息。防止请求被伪造或篡改。具体算法见下文。

注意：以上字段都在HTTP请求的参数中携带，因此最终发送时需要进行URL编码。例如： grant_type=app_token&client_id=288XXXX407&api_key=uM%2F4yRvaNQtv3tlpu%2FhL8fSTWhw%3D&device_id=8888XXX666&md5_sign=e9%3Ae1%3Afe%3Adf%3Adc%3A75%3Ac6&sha256_sign=dd%3A37%3A5e%3Aac%3A49&signature=471dfe0ffb26539a372400f82ab84b6906d70dab144b700d813bbdc9d63f2806d0a68

参数加密算法
  • API Key：小爱开放平台中分配的API Key。
  • SignSecret：小爱开放平台中分配的SignSecret。
  • NONCE8：随机8字符字符串，取值域为abcdef1234567890，即16进制数取值范围。
  • NOISE：随机字符，无意义，取值域为abcdef1234567890，即16进制数取值范围。
  • SHA1 = [SHA1(NONCE8+client_id+API Key+device_id+SignSecret):40]
  • ALDER32 = [Alder32(NONCE8+device_id):8]
  • api_key = Base64(AES256-CBC(NONCE8+API Key))
  • signature = [NOISE:3]NONCE8[NOISE:5]SHA1[NOISE:2]ALDER32[NOISE:3]
