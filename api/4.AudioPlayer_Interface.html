AudioPlayer接口

本接口基于Alexa Voice Service v20160207进行定制， 兼容AVS AudioPlayer Interface version1.0

播放指令 Play Directive

举例

消息体 Sample Message

{
  "directive" : {
    "header" : {
      "namespace" : "AudioPlayer",
      "name" : "Play",
      "messageId" : "b54bfa05-0368-47f4-955f-929198905bdd",
      "dialogRequestId" : "9628d3af-58ac-4087-82f4-4531de7d7832"
    },
    "payload" : {
      "playBehavior" : "REPLACE_ALL",
      "audioItem" : {
        "audioItemId" : "baisi:baisi_28076031",
        "stream" : {
          "url" : "http://resource-url.mp3",
          "offsetInMilliseconds" : 0,
          "progressReport" : {
            "progressReportDelayInMilliseconds" : 15000,
            "progressReportIntervalInMilliseconds" : 900000
          },
          "token" : "baisi_28076031"
        }
      }
    }
  }
}

二进制音频附件 Binary Audio Attachment

Content-Type: application/octet-stream
Content-ID: {{Audio Item CID}}

{{BINARY AUDIO ATTACHMENT}}

参数描述

参数名 说明 数据类型
messageId 用于表示特定消息的唯一ID string
dialogRequestId 与下发指令相关联的唯一ID string
playBehavior 播放方式，目前方式有：REPLACE_ALL, ENQUEUE
REPLACE_ALL 表示替换当前播放列表中的所有音频流
ENQUEUE 表示将音频流添加到当前播放列表		 string
audioItem 包含对音频进行描述的多项Key/Value值 object
audioItem.audioItemId 音频标识符 string
audioItem.stream 包含对音频流进行描述的多项Key/Value值 object
audioItem.stream.url 音频流的url地址 string
audioItem.stream.streamFormat 音频流的格式，目前支持：AUDIO_MPEG string
audioItem.stream.offsetInMilliseconds 时间戳，指示客户端应该从音频流的哪个位置开始播放 long
audioItem.stream.expiryTime 音频流过期时间，格式为ISO8601 string
audioItem.stream.progressReport 包含对音频流播放进程进行描述的多项Key/Value值 object
audioItem.stream.progressReport.progressReportDelayInMilliseconds 指定何时向AIVS上报ProgressReportDelayElapsed Event，以milliseconds为单位。ProgressReportDelayElapsed只能以指定的间隔发送一次。注意：某些音频内容不需要上报ProgressReportDelayElapsed Event，所以progressReportDelayInMilliseconds有可能不会出现在返回结果中 long
audioItem.stream.progressReport.progressReportIntervalInMilliseconds 向AIVS上报ProgressReportIntervalElapsed Event的时间间隔，以milliseconds为单位。开发者需以该时间间隔定期上报ProgressReportIntervalElapsed Event。注意：某些音频内容不需要上报ProgressReportIntervalElapsed Event，所以progressReportIntervalInMilliseconds有可能不会出现在返回结果中 long
audioItem.stream.token 表示当前音频流的 标识 string

PlaybackStarted Event

举例

消息体 Sample Message

{
  "event" : {
    "header" : {
      "namespace" : "AudioPlayer",
      "name" : "PlaybackStarted",
      "messageId" : "225f1c3d-6f60-48c7-9d2f-705a2da79249"
    },
    "payload" : {
      "token" : "baisi_28076031",
      "offsetInMilliseconds" : 0
    }
  }
}

参数描述

参数名 说明 数据类型
messageId 用于表示特定消息的唯一ID string
token Play Directive提供的标识符 string
offsetInMilliseconds 标识音频流的偏移量，上报的该值必须大于等于0 long
