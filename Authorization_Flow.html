设备OAuth授权流程

使用Xiao AI Voice Service 服务需要小米账号授权。小米帐号服务基于 OAuth2.0 协议的开放授权和鉴权服务。本文描述典型的申请小米账号授权流程和使用方维护授权信息的处理。

前置条件

请确认您已接入小米账号。暨：

  1. 在小爱开放平台成功注册企业账号。
  2. 已经成功创建智能设备。
  3. 有自己的后台服务，并将后台服务的回调地址配入设备安全配置中的的填写授权回调地址。针对回调地址的详细配置说明，可以参考账号接入部分。
  4. 联系小米工程师开启相关权限。

授权流程

  1. 用户启动车辆。
  2. 车辆连接车厂后台服务，申请授权信息。需要携带车辆DeviceId。
  3. 车厂后台服务发现车辆没有授权信息，则构造申请授权请求的地址。地址中应当包括小爱开放平台中注册的Client ID、回调地址以及车场分配的车辆唯一设备Id。此外应当携带response_type、pt和state字段。详见后面申请授权码部分。通过二维码的方式返回授权地址，要求进行授权
  4. 车辆展示二维码。
  5. 用户打开手机应用（支付宝、微信等，后期会支持米家）。
  6. 扫描车辆二维码。
  7. 应用访问二维码中的授权地址向小米账号服务授权端点申请授权。
  8. 小米账号服务验证授权请求中的参数。Client ID，回调地址与配置相同，且state字段验证通过。
  9. 小米账号服务验证通过后返回小米账号登录页面(事实上是通过302下发重定向地址到登录页面)。
  10. 用户在手机应用输入账号信息。
  11. 账号信息直接发送到小米账号服务。（此操作由下发的登录页面完成）
  12. 小米账号服务验证用户账号信息。
  13. 验证账号信息成功后，按照第7步中携带的回调地址，把授权码和state拼接到url的参数中。通过302把回调地址发送回客户端。
  14. 客户端按照302，重定向访问回调地址。
  15. 车厂回调服务器收到请求后，向小米账号令牌中心申请令牌。请求中应当携带Client ID、回调地址、授权码、grant_type、scope_data、pt以及小爱开发平台中分配的Client Secret。详见后面申请令牌部分。
  16. 小米账号令牌中心验证请求中的信息。
  17. 小米账号令牌中心验证通过后，返回接入令牌及刷新令牌。
  18. 车厂回调服务收到令牌后，向手机应用返回授权成功。
  19. 同时车厂回调服务应当向车厂后台服务同步令牌。
  20. 车厂后台服务收到令牌后通知车辆授权成功。
  21. 车辆上显示授权成功。
  22. 车厂后台服务应当定时维护接入令牌和刷新令牌。保证令牌有效。

申请授权码接口

请求

GET  /oauth2/authorize
Host https://account.xiaomi.com

client_id=4066XXX8888&redirect_uri=http://demo.dummy.com/oauth/xiaomi&response_type=code&device_id=8888XXX666&state=68b5d3fac9d5b7b3e1a4e934f286b051bac54426&pt=1

client_id：小爱开放平台中注册的Client ID。

redirect_uri： 小爱开放平台中配置的回调地址。申请授权码成功后，授权码会下发到此地址。

response_type：应当填写为code，表明申请授权码。

device_id：设备的硬件信息标示，应当保证唯一。

pt:平台参数，1表示小爱开放平台。

state：在 oauth 2.0 协议中，state 的值是一个随机字符串，并且可以选择性的传递。在本接口中，我们对该字段进行了扩展，用来携带获取令牌接口中需要携带的 scope_data 参数的指纹信息，防止 scope_data 参数值被伪造或更改。计算方式如下：

  1. 约定获取令牌接口中的 scope_data 按照 json 格式组织，在计算 state 时需要对 json 中的元素按照 key 的字典顺序组织。
  2. 并将所有的 “key=value” 键值对通过 “&” 连接。
  3. 对得到的字符串执行 SHA1 计算，得到 state 的值。

后台会记录 state 到授权码中。

响应

如果授权成功，授权服务器会重定向请求到 redirect_uri 所指定的地址，并带上相关参数：

http://demo.dummy.com/oauth/xiaomi?code=CODE&state=STATE

申请令牌接口

请求

GET or POST  /oauth2/token
Host https://account.xiaomi.com

client_id=4066XXX8888&redirect_uri=http://demo.customer.com/oauth/xiaomi&client_secret=7659FHN_XXX_XX59&grant_type=authorization_code&code=1F832XXX666&scope_data=68b5d3fac9d5b7b3e1a4e934f286b051bac54426&pt=1

client_id：小爱开放平台中注册的Client ID。

redirect_uri： 小爱开放平台中配置的回调地址。

client_secret：小爱开放平台中分配的Client Secret。

grant_type：应当填写为authorization_code，表明申请令牌。

code：申请到的授权码。

pt：平台参数，1表示小爱开放平台。

scope_data：扩展数据，最终会被封装到访问令牌中，按 json 格式组织，然后进行 base64 编码。需要把 device_id以 “d” 为 key，记录到本字段中。 注意：如果申请授权码时传递了 state 值应当与此时传递的 scope_data对应，小米账号服务会按照相同的方式计算得到对应的签名并验证。

响应

如果授权成功，会把令牌以JSON的形式返回。

{
 "access_token": "access token value",
 "expires_in": 7200,
 "refresh_token": "refresh token value",
 "scope": "scope value",
 "token_type ": "mac",
 "mac_key ": "mac key value",
 "mac_algorithm": "HmacSha1",
 "openId":"32位字符串"
}

返回响应中包括接入令牌以及刷新令牌。接入令牌过期前车厂服务端应当使用刷新令牌获得新的接入令牌。
